# Защита данных и развёртывание

## Переменные окружения (сервер)

| Переменная | Назначение |
|------------|------------|
| `SESSION_SECRET` | Секрет подписи сессии (≥32 символов), обязателен в production |
| `HOCKEY_DATA_PATH` | Полный путь к `data.json` вне общедоступного веб-каталога (по умолчанию `server/data.json`) |
| `HOCKEY_ADMIN_PATH` | Путь к `admin.json` (по умолчанию рядом с данными) |
| `ENABLE_HSTS` | `1` — включить заголовок Strict-Transport-Security (наряду с HTTPS на прокси) |
| `TRUST_PROXY_HOPS` | Число прокси перед Node для корректного IP (по умолчанию `1`) |
| `COOKIE_SECURE` | `1` — помечать session-cookie как Secure вне production (в production включается автоматически) |
| `CORS_ORIGIN` | Явный origin для CORS с `credentials` (если фронт на другом домене) |
| `DEVICE_USER_LOG_RETENTION_DAYS` | Срок хранения записей журнала устройств/IP (по умолчанию `90`) |
| `AUTH_RATE_LIMIT_MAX` | Макс. попыток входа/регистрации за 15 минут с одного IP (по умолчанию `40`) |
| `FORGOT_PASSWORD_RATE_LIMIT_MAX` | Макс. запросов «забыли пароль» за час с одного IP (по умолчанию `8`) |
| `PUBLIC_APP_URL` | Базовый URL сайта с HTTPS (без слэша в конце), нужен для ссылки в письме сброса пароля |
| `YOOKASSA_WEBHOOK_IP_ALLOWLIST` | Опционально: список IP через запятую; если задан, только эти адреса могут вызывать HTTP-уведомления оплаты. Пусто — проверка отключена (основная гарантия — запрос платежа по API с секретом магазина) |
| `SMTP_HOST` | SMTP-хост (например `smtp.timeweb.ru`) |
| `SMTP_PORT` | Порт (часто `465` для SSL) |
| `SMTP_SECURE` | `1` или `true` для SSL на 465; для 587 с STARTTLS часто `0` и отдельная настройка в nodemailer |
| `SMTP_USER` | Логин SMTP (полный email ящика) |
| `SMTP_PASS` | Пароль ящика |
| `MAIL_FROM` | Адрес отправителя (если не задан — берётся `SMTP_USER`) |
| `MAIL_SITE_NAME` | Название в теме письма (по умолчанию `Hockey Tactics`) |
| `SMTP_DEBUG` | `1` — подробный лог SMTP в консоль Node (только на время отладки) |

Если письмо не уходит: в логе сервера смотрите строку `[mail] forgot-password:` (код ответа SMTP). Частые причины: в `.env` указан **не тот пароль** (оставили текст «ваш_пароль» из примера вместо реального пароля ящика), неверный порт/шифрование (для Timeweb обычно `465` и `SMTP_SECURE=1`), блокировка исходящего `465` сетью. Альтернатива: порт `587`, `SMTP_SECURE=0`, в некоторых конфигурациях добавляют `SMTP_SECURE=starttls` (см. актуальную справку хостинга).

## Резервное копирование

Файлы `data.json` и `admin.json` содержат персональные и служебные данные. Храните копии вне сервера с ограниченным доступом; при передаче в облако шифруйте архив (например, `gpg` или средствами провайдера).

Пример (локально, после остановки сервера или с копией файла):

```bash
gpg --symmetric --cipher-algo AES256 -o backup-data.json.gpg /path/to/data.json
```

## Продакшен

- Только **HTTPS**, редирект с HTTP, при необходимости HSTS на reverse-proxy и/или `ENABLE_HSTS=1`.
- Не выкладывайте `data.json` в каталог статики и в git.
- Права ОС: чтение/запись данных только у пользователя процесса Node.
