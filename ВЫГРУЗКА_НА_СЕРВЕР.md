# Выгрузка на сервер (пользователи сохраняются)

**Ваш сервер:** `72.56.233.203`  
**Папка проекта на сервере:** `/root/hockey`  
**Папка проекта на вашем Mac:** `/Users/test/Hockey`

Пользователи и данные лежат в файле **`server/data.json` на сервере**. Скрипт выгрузки его **не перезаписывает**. Локальный `data.json` с Mac на сервер не подменяет.

### Быстрая выгрузка без затрагивания пользователей

Используйте эту последовательность (на Mac, в терминале):

```bash
cd /Users/test/Hockey
export DEPLOY_SSH='root@72.56.233.203'
export DEPLOY_PATH='/root/hockey'
npm run deploy:server
```

Это подключает `scripts/rsync-to-server.sh`: бэкап `data.json`/`admin.json` на сервере, синхронизация кода без подмены этих файлов, `npm ci`/`npm install` при необходимости и перезапуск приложения.

## Как выгрузить (автоматически)

Все команды выполняете **на Mac**, в терминале.

**1.** Перейдите в проект:

```bash
cd /Users/test/Hockey
```

**2.** Укажите сервер и каталог (можно один раз скопировать целиком):

```bash
export DEPLOY_SSH='root@72.56.233.203'
export DEPLOY_PATH='/root/hockey'
```

**3.** Запустите выгрузку:

```bash
npm run deploy:server
```

Скрипт сам: соберёт сайт (`dist/`), сделает на сервере копию `data.json` и `admin.json` с меткой времени, зальёт код **без** замены этих файлов на сервере, обновит зависимости и перезапустит приложение (`pm2 restart hockey`).

**4.** Проверка (по желанию):

```bash
ssh root@72.56.233.203
cd /root/hockey
pm2 logs hockey --lines 30
exit
```

Откройте сайт в браузере и проверьте вход пользователя.

## Ручная выгрузка (без скрипта)

**На сервере — бэкап:**

```bash
ssh root@72.56.233.203
cd /root/hockey
cp server/data.json server/data.json.bak.manual
cp server/admin.json server/admin.json.bak.manual
exit
```

**На Mac:**

```bash
cd /Users/test/Hockey
npm run build
```

**Заливка** (не копирует `data.json` и `admin.json` с Mac на сервер):

```bash
rsync -avz --delete ./dist/ root@72.56.233.203:/root/hockey/dist/
rsync -avz \
  --exclude 'data.json' \
  --exclude 'admin.json' \
  --exclude 'uploads' \
  --exclude '*.bak' \
  ./server/ root@72.56.233.203:/root/hockey/server/
scp package.json package-lock.json root@72.56.233.203:/root/hockey/
```

**На сервере:**

```bash
ssh root@72.56.233.203
cd /root/hockey
npm ci --omit=dev
pm2 restart hockey
exit
```

## Если что-то сломалось

На сервере можно вернуть старый файл пользователей из бэкапа (имя файла посмотрите в папке `server/`):

```bash
ssh root@72.56.233.203
cd /root/hockey
ls server/data.json*
# пример:
# cp server/data.json.bak.202603301200 server/data.json
pm2 restart hockey
```

Логи: `pm2 logs hockey`

Файл **`.env` на сервере** при обычной выгрузке не трогают — не заливайте поверх него случайно локальный `.env`, если настройки отличаются.

## Тарифы и пользователи

- **Пользователи** хранятся в **`server/data.json` на сервере**. Скрипт деплоя его **не перезаписывает** — записи и тарифные поля в профилях сохраняются.
- **Лимиты и описания тарифов** задаются в коде: **`server/tariffLimits.js`**, **`server/tariffs.js`** и зеркально во фронте **`src/constants/tariffLimits.js`**, **`src/constants/tariffs.js`**. При `rsync` на сервер уходят **локальные** файлы из `server/` — правки **только на сервере** при следующем деплое **затрутся**, если их не перенести в репозиторий на Mac.

Если вы меняли тарифы **на сервере**, перед выгрузкой:

1. Скопируйте файлы с VPS и сравните с проектом на Mac:
   ```bash
   scp root@72.56.233.203:/root/hockey/server/tariffLimits.js /tmp/tariffLimits.server.js
   scp root@72.56.233.203:/root/hockey/server/tariffs.js /tmp/tariffs.server.js
   diff -u server/tariffLimits.js /tmp/tariffLimits.server.js
   diff -u server/tariffs.js /tmp/tariffs.server.js
   ```
2. Внесите нужные отличия в **`/Users/test/Hockey/server/tariffLimits.js`** и **`server/tariffs.js`**, затем синхронизируйте лимиты и формулировки с **`src/constants/tariffLimits.js`** и **`src/constants/tariffs.js`** (лимиты по тарифам должны совпадать между серверным и клиентским `tariffLimits`).
3. Закоммитьте изменения и выполните **`npm run deploy:server`**.

Либо можно один раз запустить скрипт сравнения (из корня проекта):

```bash
bash scripts/diff-server-tariffs.sh
```

(нужен рабочий `ssh`/`scp` к серверу с вашего Mac.)
