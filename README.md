# Hockey Tactics — План-конспекты для хоккеистов

Платформа для создания тактических схем и план-конспектов на хоккейной площадке.

## Запуск

**Вариант 1 — один сервер (рекомендуется):**
```bash
npm install
npm run build
npm run server
```
Откройте http://localhost:3002

**Вариант 2 — режим разработки (два терминала):**
```bash
npm run server   # терминал 1 — API на порту 3002
npm run dev      # терминал 2 — фронтенд на порту 5173
```
Откройте http://localhost:5173

## Обновление на хостинге (не терять пользователей)

Зарегистрированные пользователи, планы и доски лежат в **`server/data.json`**. Настройки админки и контент страниц — в **`server/admin.json`**.

При выкладке новой версии:

1. **Перед обновлением** сделайте копию на сервере:  
   `cp server/data.json server/data.json.bak` (и при необходимости `admin.json`).
2. Обновите код (`git pull` или залейте архив), установите зависимости и соберите фронт:  
   `npm ci --omit=dev` (или `npm install --omit=dev`), затем `npm run build`.
3. **Не подменяйте** рабочие `server/data.json` и `server/admin.json` файлами из архива разработки, если там пустая база.
4. Перезапустите процесс (`pm2 restart …`, `systemctl restart …` или как у вас настроено).

Если деплой через «залить всю папку по FTP»: **не загружайте поверх** существующие `data.json` / `admin.json` на сервере — обновляйте только код, `dist/`, `package.json`, `server/index.js` и остальные файлы приложения **кроме** этих двух JSON.

В репозитории эти файлы в **`.gitignore`**, чтобы случайно не затереть продакшен-данные коммитом с локальной машины.

## Функционал

- **Регистрация** по email и паролю (логин = часть до @)
- **Простая капча** при регистрации (решение примера)
- **Админ**: логин `myadmin`, пароль `gjf25hortF#`
- **Тактическая доска** с подложкой хоккейной площадки
- **Инструменты**: карандаш, линия, стрелка, прямоугольник, круг, ластик, иконки игроков и шайбы
- **Перемещение** иконок мышью
- **Масштаб** — колёсико мыши
- **Панорама** — Alt + перетаскивание

## Конвертация в приложение

Проект на React + Vite готов к конвертации в мобильное приложение через [Capacitor](https://capacitorjs.com/) (iOS/Android).
