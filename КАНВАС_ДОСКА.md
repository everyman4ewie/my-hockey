# Как устроена отрисовка на тактической доске (на сайте)

Речь о том, **как рисунок существует в приложении**, а не о формате скачивания (PNG и т.п.).

## Что видит пользователь в браузере

Используется стандартный **HTML5 `<canvas>`** с контекстом **`2d`** (`getContext('2d')`). То есть изображение на доске — это **растровая отрисовка в буфер канваса** (пиксели по ширине и высоте элемента). При изменении данных сцена **перерисовывается** кодом (фон льда, линии, иконки и т.д.).

## В каком виде хранятся данные рисунка

В памяти React это **структуры JavaScript**: массивы **`paths`** (линии, кривые, стрелки, фигуры) и **`icons`** (игроки, шайбы, ворота и т.д.) с координатами в **логической системе пикселей канваса** (ось X вправо, Y вниз).

Размеры буфера канваса задаются пропами компонента доски; **по умолчанию** — **800×400** пикселей (полный лёд; для других зон поля могут быть другие размеры).

## Кратко

| Аспект | Описание |
|--------|----------|
| Отрисовка на экране | Canvas 2D API (растр в окне браузера) |
| Модель данных | Объекты с координатами и параметрами (не SVG-файл и не PNG, пока не экспортируете) |
| Скачивание | Отдельно: обычно растр через `canvas` (например, PNG) — это уже другой шаг |

Файл можно дополнять по мере появления новых зон поля или режимов масштабирования.

## Каталог готовых упражнений

На сайте есть **отдельная сущность «каталог»** (админ заполняет записи с массивом `exercises`, как в план-конспекте). Пользователь **никогда не правит оригинал каталога**: при добавлении в свой план или на тактическую доску создаётся **полная копия** данных (с новыми id у путей, иконок и слоёв, чтобы не было коллизий). У копии в плане может быть метка источника и кнопка **просмотра оригинала** в режиме только чтения. Слои в упражнении плана поддерживаются **только если** они заданы в шаблоне каталога; иначе используется одно полотно `canvasData`, как раньше.
